# Análise de nLLMs - one-llm-4-all

**Data:** 02/02/2026 17:35
**Objetivo:** Identificar e documentar todas as implementações nativas de LLMs (nLLMs) presentes no sistema

---

## 📊 Resumo Executivo

O sistema **one-llm-4-all** possui **3 (três) implementações nativas de LLMs** (nLLMs), todas localizadas na pasta `/packages`:

1. **OpenAI Client** (`/packages/openai`)
2. **Anthropic Client** (`/packages/anthropic`)
3. **Google GenAI Client** (`/packages/google-genai`)

---

## 🔍 Detalhamento das nLLMs

### 1. OpenAI Client (`/packages/openai/index.ts`)

**Descrição:** Implementação nativa compatível com a API da OpenAI.

**Características:**
- ✅ Suporte a streaming
- ✅ Formato JSON (`response_format: { type: 'json_object' }`)
- ✅ Configuração de `baseURL` customizável
- ✅ Parâmetros: `temperature`, `max_tokens`, `top_p`, `stop`

**Provedores suportados via compatibilidade OpenAI:**
- OpenAI (`https://api.openai.com/v1`)
- Groq (`https://api.groq.com/openai/v1`)
- OpenRouter (`https://openrouter.ai/api/v1`)
- DeepSeek (`https://api.deepseek.com`)
- Mistral AI (`https://api.mistral.ai/v1`)
- Perplexity (`https://api.perplexity.ai`)

**Métodos principais:**
```typescript
async createChatCompletion(params: ChatCompletionParams)
private async createChatCompletionStream(params: ChatCompletionParams)
```

---

### 2. Anthropic Client (`/packages/anthropic/index.ts`)

**Descrição:** Implementação nativa para Claude (Anthropic).

**Características:**
- ✅ Suporte a mensagens multi-turn
- ✅ System prompt dedicado
- ✅ Versão da API configurável (padrão: `2023-06-01`)
- ✅ Stop sequences
- ✅ Streaming (via `parseAnthropicStream`)

**Endpoint:**
- `https://api.anthropic.com/v1`

**Métodos principais:**
```typescript
async createMessage(params: MessagesParams)
```

**Headers obrigatórios:**
- `x-api-key`: Chave da API
- `anthropic-version`: Versão da API

---

### 3. Google GenAI Client (`/packages/google-genai/index.ts`)

**Descrição:** Implementação nativa para Google Gemini.

**Características:**
- ✅ `generationConfig` com `temperature`, `topP`, `maxOutputTokens`
- ✅ `systemInstruction` para prompts de sistema
- ✅ Formato JSON via `responseMimeType: 'application/json'`
- ⚠️ **Streaming não implementado** (apenas texto e JSON)

**Endpoint:**
- `https://generativelanguage.googleapis.com/v1beta/models`

**Métodos principais:**
```typescript
async generateContent(params: GenerateContentParams)
```

**Observação:**
- A API key é passada como query parameter (`?key=${apiKey}`)

## 🏗️ Arquitetura Unificada

Todas as 3 nLLMs são utilizadas pelo **adaptador unificado** em `/src/adapters/run-llm.ts`:

```typescript
export async function runLLM(
  params: UnifiedLLMParams,
  output: "text" | "json" | "stream",
): Promise<UnifiedLLMTextResult & { json?: unknown; stream?: AsyncGenerator<string> }>
```

### Fluxo de Decisão

```
runLLM()
  ├─ OpenAI Compatible? → OpenAIClient
  │   ├─ openai
  │   ├─ groq
  │   ├─ openrouter
  │   ├─ deepseek
  │   ├─ mistral
  │   └─ perplexity
  │
  ├─ anthropic? → AnthropicClient
  │
  └─ gemini? → GoogleGenAIClient
```

---

## 🧩 Dependências Nativas

O sistema utiliza a biblioteca **reqify** (`/packages/reqify`) como camada de HTTP nativa, evitando dependências externas como `axios`.

**Métodos utilizados:**
- `reqify.post(url, data, options)` - Requisições normais
- `reqify.postStream(url, data, options)` - Requisições com streaming

## 📝 Interface Fluente

As 3 nLLMs são acessadas através da **Fluent Interface** em `/src/fluent/send-prompt.ts`:

```typescript
import { sendPrompt } from 'one-llm-4-all';

// Exemplo com OpenAI Client (via Groq)
const txt = await sendPrompt('Explique LLMs', {
  model: 'llama-3.1-8b-instant',
  provider: 'groq'
}).getText();

// Exemplo com Anthropic Client
const response = await sendPrompt('Analise este código', {
  model: 'claude-3-opus-20240229',
  provider: 'anthropic'
}).getText();

// Exemplo com Google GenAI Client
const json = await sendPrompt('Retorne JSON', {
  model: 'gemini-pro',
  provider: 'gemini'
}).getJSONResponse();
```
